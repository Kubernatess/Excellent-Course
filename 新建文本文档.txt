返回集一样的查询语句能否合并

字符串用Null表示没有,数字类型用-1或者0表示没有。如果要表示空,null明显语义比空串要好很多。Null、空串都可以表示空的意义,这个还得看环境的默认设置。如果字段不能为空串,这个只能由程序来控制。如果是枚举类型的字段,可以给它们设置默认值。tabIndex也是枚举类型,但是如果设置了默认值为A,就有可能占用了别人的主键


user表
identity 必填项,不能出现空串,也没有默认值
status 必填项,不能出现空串,枚举类型可以设置默认值
password 必填项,不能出现空串,也没有默认值
name 必填项,不能出现空串,也没有默认值
depart 必填项,不能出现空串,枚举类型可以设置默认值

course表
id 必填项,不能出现空串,也没有默认值
teacher_identity 必填项,不能出现空串,而且必须是老师的id,也没有默认值
name 必填项,不能出现空串,也没有默认值
description 必填项,不能出现空串,也没有默认值
team 必填项,不能出现空串,也没有默认值
major 选填项,可以使用空串来表示没有,选填项可以直接用没有作为默认值

tab表
course_id 必填项,属于外键,同时属于组合主键,不能出现空串,也没有默认值
index 必填项,不能出现空串,也没有默认值
name 必填项,不能出现空串,默认值为'新建选项卡'

column表
course_id 必填项,不能出现空串,没有默认值
tab_index 必填项,不能出现空串,没有默认值
index 必填项,为数字类型,这里不能出现0,也没有默认值
name 为必填项,也不能出现空串,默认值为'新建栏目'
content 选填项,还可以接收空串,选填项可以用Null作为默认值
answer 选填项,可以使用空串来表示没有,用空串作为默认值。这里答案数据可以频繁修改

record表
course_id 必填项,不能出现空串,没有默认值
tab_index 必填项,不能出现空串,没有默认值
column_index 必填项,为数字类型,这里不能出现0,也没有默认值
student_identity 必填项,不能出现空串,而且必须是学生的id,也没有默认值
hitcount 必填项,默认值为1,意味着起始访问量为1
score 为选填项,因为可以接收0分,所以用-1表示没有的意思,所以用-1来表示默认值。这个分数只能添加一次,一旦有了分数后,就不得更改
finished 为必填项,布尔类型,一旦设置为true,就不允许再更新,默认值为0

comment表
identity 必填项,可以是老师工号也可以是学生学号,不允许空串,没有默认值
date_time 必填项,也不能出现空串,没有默认值
course_id 必填项,课程评价、课程讨论、回复评论都用到这个字段,不接受空串,没有默认值
tab_index 选填项,只有在课程讨论、回复评论才会用到这个字段,不接收空串,所以可以用空串作为默认值
column_index 选填项,只有在课程讨论、回复评论才会用到这个字段,不接收0,所以可以用0作为默认值
opposite 选填项,只有在回复评论才会用到这个字段,表示回复那个对方的id,同样不接受空串,所以可以用空串作为默认值。但是因为引用了外键,所以只能用Null来作为默认值
lunch_time 选填项,只有在回复评论才会用到这个字段,表示对方发起评论的时间,这个是一个定位字段数据。同理,它也是引用了外键,所以只能用Null作为默认值
comment 必填项,不接受空串,没有默认值
